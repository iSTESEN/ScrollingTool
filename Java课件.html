容器

Java的容器类库主要分为两大类：集合（Collection）框架和映射（Map）接口。它们都位于java.util包下，是Java中数据结构和算法的重要组成部分。以下是对Java容器的详细介绍：

Collection框架

Collection框架是一个用于表示和操作集合的统一架构，它包含了几种主要的接口和其实现类，主要有以下几类：

List（列表）：

  • 接口：List，有序且可重复。
  • 实现类：ArrayList（动态数组，查询快，增删慢）、LinkedList（双向链表，增删快，查询慢）、Vector（线程安全的ArrayList，较老，使用较少）。

Set（集）：

  • 接口：Set，无序且不允许重复。
  • 实现类：HashSet（基于哈希表，查询速度快）、LinkedHashSet（保持插入顺序的Set）、TreeSet（基于红黑树，自然排序或自定义排序）。

Queue（队列）：

  • 接口：Queue，先进先出（FIFO）的数据结构。
  • 实现类：LinkedList（也可以作为队列使用）、ArrayDeque（双端队列，高效）、PriorityQueue（优先队列，按优先级排序）。

Deque（双端队列）：

  • 接口：Deque，双端队列，允许两端添加和移除元素。
  • 实现类：ArrayDeque、LinkedList。

Map（映射）：

虽然Map不是Collection的一部分，但经常与Collection一起讨论，因为它也是重要的数据结构。

  • 接口：Map，存储键值对。
  • 实现类：HashMap（基于哈希表，查询速度快）、LinkedHashMap（保持插入顺序或访问顺序）、TreeMap（基于红黑树，自然排序或自定义排序）、Hashtable（线程安全的HashMap，较老，使用较少）、ConcurrentHashMap（线程安全的HashMap，高效）。

特殊容器

Stack：继承自Vector，实现了栈数据结构，但通常推荐使用Deque作为栈，因为它更灵活且功能更多。

Collections：这不是一个容器，而是一个包含了许多静态方法的工具类，用于操作集合，如排序、查找最大最小值、批量设置不可变等。

新的容器类（Java 8及以上）

Stream：虽然不是一个传统意义上的容器，但它提供了一种高效、易于并行处理的高级数据处理方式，常用于集合的复杂操作。

2、实现Runnable接口

  • 由于java只允许单继承，如果一个类的继承别有他用，那此时就被占用了，得不尝失
  • Runnable接口内，只有一个run()方法需要实现
// FunctionalInterface函数式接口，内只有一个方法
@FunctionalInterface
public interface Runnable {
    public abstract void run();
}
  • 实现了Runnable接口的类对象，可以通过Thread类代理运行线程
  • 使用Runnable接口的方法，可以更方便的访问共享资源
// 【1】实现Runnable接口，并重写run()方法，让本类有可以线程运行的入口
public class FF2 implements Runnable {
    private int total = 10;


    public static void main(String[] args) throws IOException {
        FF2 f = new FF2();

        // 【3】通过Thread类，放入Runnable接口实现类对象，代理该对象运行线程
        Thread t1 = new Thread(f);
        Thread t2 = new Thread(f);
        Thread t3 = new Thread(f);
        t1.start();
        t2.start();
        t3.start();
        
        System.out.println("Main结束");
    }

    @Override
    public void run() {
        while (true) {
            // 【2】通过同步锁，锁定每次只有唯一一个对象访问total
            synchronized(this) {
                if(total>0) {
                    System.out.println("剩余：" + --total);
                    try {
                        Thread.sleep(600);
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                } else {
                    return;
                }
            }
        }
    }
    
}

3、实现Callable接口

  • 使用Runnable虽然挺灵活，但是缺点是不能获取线程返回值
  • Callable接口与Runnable实现方式类似，其是实现了一个V call()方法
@FunctionalInterface
public interface Callable<V> {
    V call() throws Exception;
}
  • Thread类不支持直接通过Callable实现类启动线程，需要通过FutureTask类进行包装后才可以用Thread启动线程
  • FutureTask可以get()Callable的返回值
// 【1】实现Callable<V>接口，并重写call()方法
public class FF2 implements Callable<String> {
    private int total = 10;


    public static void main(String[] args) throws IOException, InterruptedException, ExecutionException {
        FF2 f = new FF2();

        // 【2】用FutureTask<V>包含Callable实现类对象，以便通过FutureTask获取线程返回值
        FutureTask<String> ftask1 = new FutureTask<>(f);
        Thread t1 = new Thread(ftask1);
        FutureTask<String> ftask2 = new FutureTask<>(f);
        Thread t2 = new Thread(ftask2);
        FutureTask<String> ftask3 = new FutureTask<>(f);
        Thread t3 = new Thread(ftask3);

        // 【3】运行线程
        t1.start(); t2.start(); t3.start();

        System.out.println("main结束");

        // 【4】使用.get()可以阻塞等待线程返回，futureTask.get(long timeout, TimeUnit unit)可以设置超时
        //     futureTask.isDone()来检查任务是否已完成
        System.out.println(ftask1.get());
        System.out.println(ftask2.get());
        System.out.println(ftask3.get());
    }


    @Override
    public String call() throws Exception {
        while (true) {
            synchronized(this) {
                if(this.total>0) {
                    System.out.println("剩余：" + --this.total);
                } else {
                    return "卖完了";
                }
            }
        }
    }

   
    
}

Throwable

java.lang.Throwable有两个子类：Error、Exception

  • Error：Java虚拟机错误，属于系统错误，很少见，免检异常
  • Exception：异常，由程序和外部环境导致的
    • ClassNotFoundException：必检异常
    • IOException：必检异常
    • RuntimeException：NullPointerException 等运行时异常，属于免检异常

  • 必检异常：在编译时检查的异常，该异常要求修正，否则无法编译
  • 免检异常：在编译时不检查的异常，可以用try catch捕获，但不强求捕获

  • File类可以操作文件（创建、删除、创建文件夹...），但是无法对文件内容操作（加减字符）

三、代码操作

  • 1、文件打开、判断存在、创建、删除
import java.io.File;
import java.io.IOException;

public class FF1 {
    public static void main(String[] args) throws IOException, InterruptedException {
        // 【1】 创建File对象，指明一个存在或不存在的文件地址
        File file = new File("." + File.separator + "smile1.txt"); 
                            // Windows: .\\smile1.txt
                            // Linux:   ./smile1.txt

        // 【2】file.exists() 判断文件是否存在
        if(file.exists()) {
            System.out.println("文件存在");
        } else {
            System.out.println("文件不存在，创建文件");
            // 【3】file.createNewFile() 创建文件，若文件已存在，则无效
            file.createNewFile();
        }
        Thread.sleep(2000);
        if(file.exists()) {
            System.out.println("文件存在，删除文件");
            // 【4】file.delete() 删除文件
            file.delete();
        }
    }
}

  • 4、转换文件大小单位
import java.io.File;
import java.io.IOException;
import java.math.BigDecimal;


public class FF2 {
    @SuppressWarnings("deprecation")
    public static void main(String[] args) throws IOException, InterruptedException {
        File file = new File("." + File.separator + "a" + File.separator + "b" + File.separator + "c" + File.separator + "img.png");

        if(!file.exists()) {
            return;
        }

        // 读取文件大小并转换成MB单位
        BigDecimal size = new BigDecimal((double)file.length()/1024/1024); // 【1】将Byte转换成MByte
        size = size.divide(new BigDecimal(1), 2, BigDecimal.ROUND_HALF_UP);//【2】divide方法不会改变原数，需要改变原数
        System.out.println("文件大小：" + size + " MB");
    }
}

文件流

一、理论知识

  • 1、文件流又分为字节流、字符流，二者操作方式不一
    • 上两行是字节流，下两行是字符流
    • 最上层的InputStream、OutputStream、Writer、Reader都是抽象类
    • 一般用中间的类进行操作
    • 字节流默认不用缓冲，字符流默认使用缓冲，导致字符流需要flush()或close()才会生效操作

二、字节流

1、FileOutputStream 输出

常用方法


public abstract class OutputStream implements Closeable, Flushable常用方法

public class FileOutputStream extends OutputStream常用方法

代码操作
  • 1、字节流写入数据到文件write()
import java.io.*;
public class FF2 {
    public static void main(String[] args) throws IOException {
        File file = new File("." + File.separator + "a" + File.separator + "output.data");

        // 创建文件夹
        if(!file.getParentFile().exists()) {
            file.getParentFile().mkdirs();
        }

        // 创建操作流
        OutputStream outs = new FileOutputStream(file); // 目标文件存在则覆盖，不存在则自动创建
        String str = "我是东西！";
        byte data[] = str.getBytes(); // 转换成字节数组
        outs.write(data);
        outs.close(); // 字节流是直接响应的，若无close也会写入
    }
}

2、FileInputStream 输入

常用方法


public abstract class InputStream implements Closeable

public class FileInputStream extends InputStream

代码操作
  • 1、读取字节数组数据read(byte[])
import java.io.*;

public class FF2 {
    public static void main(String[] args) throws IOException {
        File file = new File("." + File.separator + "a" + File.separator + "input.data");
        if(!file.exists()) {
            System.out.println("文件不存在");
            return;
        }

        InputStream iStream = new FileInputStream(file);
        byte buf[] = new byte[6]; // 此处故意取小，让起读取多次直到结束，取大也行
        int len = 0;
        // read读取到结尾会返回-1
        while((len=iStream.read(buf)) != -1) {
            System.out.print(new String(buf, 0, len));
        }
        System.out.println();
    }
}
  • 2、读取一个一个字节数据read()
public class FF2 {
    public static void main(String[] args) throws IOException {
        File file = new File("." + File.separator + "a" + File.separator + "input.data");
        if(!file.exists()) {
            System.out.println("不存在");
            return;
        }

        InputStream iStream = new FileInputStream(file);
        byte buf[] = new byte[1024];
        byte temp = 0;
        int flag = 0;
        while((temp = (byte)iStream.read()) != -1) { // 此方法一个一个字节来读，读到文件尾会返回-1
            if(flag > 1024) {break;}
            buf[flag++] = temp;
        }
        // 由于Java的字符比较特殊，遂用byte数组
        System.out.println(new String(buf, 0, flag));
    }
}

2、Reader 输入 

public class FF2 {
    public static void main(String[] args) throws IOException {
        File file = new File("." + File.separator + "a" + File.separator + "read.data");
        if(!file.exists()) {
            System.out.println("不存在");
            return;
        }

        Reader reader = new FileReader(file);
        char buf[] = new char[1]; // 没错写1难为他了，要的就是这个效果
        int len  = 0;
        while((len = reader.read(buf)) != -1) {
            System.out.print(new String(buf, 0, len));
        }
        System.out.println();
    }
}
